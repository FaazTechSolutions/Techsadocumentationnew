  • project, crud
    • project employees, create, view,
    • delete(only if valid from equals to valid to),
    • end project(valid to, comments) no validations just updating the valid to date in project employee table,
    • updating the comments in common follow-up comments table
  • customer, crud, no delete option,
    • customer details
    • project, crud
    • project employees only view and edit option
  • task, crud with attachment
    • search option (assigned to, Scheduled Start Date, Scheduled End Date)
    • tab active task list, completed task list
    • task creation:
      • creation multiple employee assignment for single task,
      • if the employee is already assigned in task he will not be available in task creation
      • updated in dependent tables like Task History, Task Assignment,
    • task update:
      • except assign to other fields can be updated,
      • no updates in dependent tables,
    • task details:
      • print option with login history and task details,
      • print option with task details,
      • task edit option,
      • assign project employees action
      • task action:
        • Action status:
          • New = 1,
          • Process = 2,
          • Hold = 3,
          • Near To Complete = 4,
          • Complete = 5,
        • Actions:
          • start, if status = 1,
          • Hold, if status = 2||4,
          • Resume, if status = 3,
          • Near To Complete, if status = 2
          • Complete, if status = 2||4
          • Update Progress, if status = 2||4
      • assignment history, tab 1
      • end employee action(validating is the employee active in task and updating the valid to date as current date and comments in task assignment table)
      • current active employees, tab 2
      • status history: view all and edit only comments
      • comments create option
  • task template: crud option to show the template for all or only for admin
  • employee, crud
    • delete validated if he is assigned In any project we don't allow to delete the record
    • employee details:
      • profile,
      • all tasks assigned to him,
      • all projects he assigned,
      • task from current project
      • terminate employee action,
        • updating the date and reason in employee table,
        • updating the termination date in project employee in field valid to
        • user login access is stopped
        • user gate status updated to inactive
        • updating the comments in common follow up comments table
